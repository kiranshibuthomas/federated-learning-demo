Training Process Timeline

Step 1: Local Training

Each user's model trains on their private data. Alex learns sports patterns, Bailey learns baking patterns, Chris learns programming patterns. Data never leaves their device.

Step 2: Extract Model Weights

Only the learned patterns (n-grams, word frequencies) are extracted. For example: "the game" → ["was", "is", "will"]. No actual sentences are shared.

Step 3: Federated Averaging

Server merges the patterns from all three users. If Alex says "the" → "game" (5 times) and Bailey says "the" → "cake" (3 times), the merged model knows both patterns.

Step 4: Distribute Global Model

The merged model is sent back to all users. Now everyone has a model that knows sports, baking, AND programming patterns.

Step 5: Local Fine-tuning

Each user adapts the global model to their preferences. Alex's model still predicts sports best, but now has some general knowledge too.

Privacy: What Gets Shared vs What Stays Private

✓ Shared (Safe)

  • N-gram patterns: "the game" → "was"
  • Word frequencies: "game" appears 15 times
  • Bigram statistics: "football" → "match"
  • Model accuracy: 75%
  • Training loss: 0.35

✗ Never Shared (Private)

  • Raw sentences: "the game was amazing"
  • User messages: "I love football"
  • Search queries: "best basketball shoes"
  • Typing history: All keystrokes
  • Personal information: Any PII

Model Architecture

Each user has an N-gram language model that learns word patterns from their typing history.

Component Description Example
Trigram Model Predicts next word based on last 2 words "the game was" → "amazing"
Bigram Model Predicts next word based on last 1 word "game" → "was"
Word Frequency Fallback to most common words "the", "is", "was"
Vocabulary Size Number of unique words per user ~200 words
Training Data Number of sentences per user 140+ sentences

Example: What the Server Sees

When Alex's model sends updates to the server, here's what gets transmitted:

{ "ngramModel": { "the game": {"was": 5, "is": 3, "will": 2}, "football match": {"postponed": 2, "scheduled": 3, "won": 1} }, "bigramModel": { "game": {"was": 8, "is": 5, "will": 3}, "team": {"won": 6, "lost": 4, "played": 5} }, "wordFrequency": { "the": 45, "game": 15, "team": 12, "player": 10 }, "accuracy": 0.75, "loss": 0.35 }

Notice: No actual sentences! The server can't reconstruct "the game was amazing yesterday" from these patterns.

Federated Averaging Algorithm

The server merges patterns from all users by combining their frequencies:

// Alex's model says: "the" → "game" (count: 5) // Bailey's model says: "the" → "cake" (count: 3) // Chris's model says: "the" → "code" (count: 4) // Merged global model: "the" → { "game": 5, // from Alex "cake": 3, // from Bailey "code": 4 // from Chris } // Now all users can predict all three words!

Benefits of Federated Learning

Benefit Traditional ML Federated Learning
Privacy ❌ Data sent to server ✅ Data stays on device
Personalization ❌ One model for all ✅ Personalized per user
Data Security ❌ Risk of breaches ✅ No central data store
Compliance ❌ GDPR concerns ✅ GDPR compliant
Bandwidth ❌ Send all data ✅ Send only weights (~2MB)
