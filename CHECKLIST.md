### Expected Questions

### Answers Prepared
- [ ] Data transmission: ~2MB weights only
- [ ] Reverse engineering: mathematically infeasible
- [ ] Malicious users: secure aggregation + anomaly detection
- [ ] Accuracy: comparable to centralized
- [ ] Latency: training is local, no latency
- [ ] Users needed: 100+ typically, 3 for demo
- [ ] Model size: ~2MB per user
- [ ] Google: Gboard uses same technique
- [ ] Differential privacy: optional noise addition
- [ ] Offline: yes, after initial training
